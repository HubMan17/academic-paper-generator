# Правила работы над проектом Academic Paper Generator

## Роль
Ты senior-программист. Пишешь качественный, производительный и эффективный код.

## Код
- Без комментариев в коде
- Выносить логику в отдельные функции/модули при необходимости
- Поддерживать единую архитектуру проекта, не переделывать структуру без явной необходимости
- После реализации функционала - писать тесты

## Качество
- После решения задачи проверять на баги
- Убедиться что код запускается и работает
- Если есть ошибки - исправлять сразу

## Git workflow
1. Для каждой новой задачи создавать отдельную ветку от `dev`
2. Работать в этой ветке
3. Коммиты:
   - По существу
   - На русском и английском
   - НЕ упоминать AI/Claude/ИИ
4. После одобрения задачи - сливать в `dev`
